## Why

PV data uploaders should be able to send readings with one system-specific secret and no OAuth/OIDC flow, browser session, or token-exchange implementation. The existing general API-token model is secure but does not yet provide the copy-and-use header or push-URL experience expected by embedded inverters, scripts, and home-automation systems.

## What Changes

- Add system-bound ingestion API keys that are generated by PVLog, displayed once, independently revocable/rotatable, and limited to telemetry ingestion for one system.
- Accept an ingestion key either through `x-pvlog-api-key` or as the opaque credential segment of a generated push URL.
- Provide copyable header and URL examples so simple uploaders need only send JSON to one endpoint without OAuth, OIDC, cookies, or CSRF.
- Apply the same validation, idempotency, quotas, auditing, and single/batch ingestion semantics to both credential transports.
- Prevent credential transport ambiguity and require redaction of URL/header keys from logs, traces, metrics, problem details, referrers, and audit metadata.
- Keep general bearer API tokens and interactive authentication available for other API and administration use cases.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `telemetry-ingestion`: Add simple header- or push-URL-key authentication for system-scoped single and batch ingestion.
- `identity-and-access`: Define lifecycle, storage, scope, rotation, revocation, and safe transport rules for system ingestion keys.
- `modern-rest-api`: Define the stable uploader endpoint shapes, credential precedence, and safe error behavior.

## Impact

- API authentication extraction and authorization for ingestion routes.
- Management persistence and self-service APIs for account users to issue, rename, list, regenerate/rotate, and revoke system ingestion keys.
- Telemetry single/batch route aliases and OpenAPI examples.
- Request logging, tracing, proxy guidance, rate limiting, audit redaction, and security tests.
- Administration/onboarding UI and uploader documentation for copying a header value or complete push URL.
